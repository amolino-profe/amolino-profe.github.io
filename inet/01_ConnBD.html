CREATE DATABASE CREATE TABLE AUTO_INCREMENT PRIMARY KEY VARCHAR / INT

personas.sql

Conexión PDO
config/conexion.php
new PDO() setAttribute try / catch PDOException

PDO (PHP Data Objects) es la forma moderna de conectar PHP con bases de datos. Usamos try/catch para capturar errores de conexión.

config/conexion.php
💡 ¿Por qué ERRMODE_EXCEPTION? Hace que PDO lance excepciones cuando hay errores SQL. Sin esto, los errores pasan silenciosamente y son difíciles de detectar.

Tareas de este paso

  • Crear la carpeta config/ dentro del proyecto
  • Crear el archivo conexion.php
  • Escribir el código copiándolo de la imagen (¡a mano!)
  • Verificar que no haya errores de tipeo

Clase Persona
clases/Persona.php
class public __construct() $this OOP

Una clase es un molde para crear objetos. La clase Persona tiene tres propiedades y un constructor que las inicializa.

clases/Persona.php
💡 ¿Qué hace __construct? Es el método que se ejecuta automáticamente al hacer new Persona(...). Recibe los datos y los asigna a las propiedades del objeto.

Formulario HTML
formulario.html
<form> method="POST" <input> action required

La interfaz del usuario. El atributo action indica a dónde se envían los datos y method="POST" los envía de forma segura en el cuerpo de la petición.

formulario.html
💡 ¿Por qué POST y no GET? Con GET los datos aparecen en la URL (ej. ?nombre=Juan). Con POST viajan ocultos en el cuerpo de la petición, más apropiado para formularios.

Tareas de este paso

  • Crear formulario.html en la raíz del proyecto
  • Escribir la estructura HTML completa
  • Abrir en el navegador y verificar que se vean los 3 campos
  • Identificar: ¿qué hace el atributo required?
5
Guardar en la BD
guardar.php
require_once $_POST prepare() execute() INSERT INTO new Persona()

El archivo central del sistema. Captura los datos del formulario, crea un objeto Persona y lo persiste en la base de datos usando Prepared Statements.

guardar.php
🔒 ¿Por qué Prepared Statements? Los ? en el SQL son marcadores de posición. PDO los reemplaza de forma segura, evitando SQL Injection — uno de los ataques más comunes en aplicaciones web.

Tareas de este paso

  • Crear guardar.php en la raíz
  • Escribir el código completo a mano
  • Probar: llenar el formulario y enviarlo
  • Verificar en phpMyAdmin que el registro aparece en la tabla
  • Insertar al menos 3 personas de prueba
