02 — La API

REST Countries

Vamos a usar la API pública de REST Countries. Desde que pasó a la versión v5, ya no se puede usar sin autenticación: hay que crear una cuenta gratuita en restcountries.com/sign-up para obtener una API key, y enviarla en cada pedido dentro del header Authorization.

GET https://api.restcountries.com/countries/v5?response_fields=names,flag&limit=100

Con response_fields pedimos solo los campos names y flag, igual que antes hacíamos con fields. La diferencia importante es que este endpoint devuelve páginas de países, no la lista completa de una sola vez.

Cada pedido necesita el header:

Authorization: Bearer TU_API_KEY

⚠ Paso extra: habilitar CORS para tu key

Como el fetch se hace directo desde el navegador, la API necesita saber desde qué origen se le puede llamar con esa key. Por defecto, cualquier pedido desde el navegador queda bloqueado. Hay que entrar a restcountries.com/api-keys, editar la key y en CORS Allowed Origins agregar localhost (solo el hostname, sin http:// ni puerto). Si te olvidás de este paso, el pedido va a fallar en el navegador aunque la key sea correcta.

Paginación: por qué ya no alcanza con un solo pedido

La vieja /v3.1/all devolvía los ~250 países en un único array. La v5 pagina los resultados: por defecto trae 25 países, y el plan gratuito permite subir el limit hasta 100 por pedido como máximo. Como para el juego queremos el mazo completo de países, vamos a pedir de a 100 y repetir el pedido corriendo el offset, hasta que la API nos diga que no quedan más páginas (data.meta.more en false). Esto lo resolvemos dentro de cargarPaises(), más abajo.

Formato de la respuesta

La v5 envuelve todo dentro de data: la lista de países está en data.objects, e info de paginación en data.meta. Cada país dentro de objects tiene esta estructura:

respuesta JSON — un país (dentro de data.objects)
{ "flag": { "url_png": "https://flags.restcountries.com/v5/w320/uy.png", "url_svg": "https://flags.restcountries.com/v5/svg/uy.svg", "description": "Descripción de la bandera..." }, "names": { "common": "Uruguay", "official": "Oriental Republic of Uruguay", "native": { "spa": { "official": "República Oriental del Uruguay", "common": "Uruguay" } } } }

Los nombres de los campos cambiaron: name ahora es names (plural), y flags ahora es flag (singular), con png/svg/alt renombrados a url_png/url_svg/description.

¿Cómo accedo a cada dato?

Si tenemos un país guardado en la variable pais, accedemos a sus propiedades con punto:

Expresión→Resultado
pais.names.common→"Uruguay"
pais.names.official→"Oriental Republic of Uruguay"
pais.flag.url_png→"https://flags.restcountries.com/v5/w320/uy.png"
pais.flag.url_svg→"https://flags.restcountries.com/v5/svg/uy.svg"
pais.flag.description→"The flag of Uruguay is..."
03 — JavaScript

Función: cargarPaises()

El primer paso es traer todos los países desde la API al arrancar el juego. Usamos async/await para manejar la operación asincrónica de forma clara. Como la v5 pagina de a 100 países como máximo, repetimos el pedido corriendo el offset hasta juntarlos todos. Una vez que tenemos los datos, llamamos a la siguiente función que los usa.

cargar paises función
// Tu API key de restcountries.com/sign-up const API_KEY = "TU_API_KEY"; // Variable global donde guardamos todos los países let paises = []; let paisCorrecto; async function cargarPaises() { try { let offset = 0; let hayMas = true; // La API devuelve los países de a páginas (máximo 100 por pedido) // así que repetimos el fetch hasta traerlos todos while (hayMas) { // fetch hace la petición a la API, con la API key en el header const respuesta = await fetch( `https://api.restcountries.com/countries/v5?response_fields=names,flag&limit=100&offset=${offset}`, { headers: { "Authorization": `Bearer ${API_KEY}` } } ); // Convertimos la respuesta a JSON const datos = await respuesta.json(); // Los países de esta página están en datos.data.objects paises = paises.concat(datos.data.objects); // datos.data.meta.more indica si quedan más páginas hayMas = datos.data.meta.more; offset += 100; } console.log("Países cargados:", paises.length); // Una vez cargados, mostramos la primera pregunta mostrarPaisAleatorio(); } catch (error) { console.log("Error al cargar países:", error); } }

El try/catch nos permite manejar errores de red o de la API sin romper la aplicación. Si algo falla, lo mostramos en consola. Ojo con la API_KEY hardcodeada: sirve para este tutorial, pero en un proyecto real conviene no subirla a un repositorio público.

Preloader: ahora tarda más

Como cargarPaises() hace varios pedidos en secuencia (uno por página) en vez de uno solo, la carga inicial tarda un poco más que con la v3.1. Conviene mostrar algún indicador de carga (un spinner, por ejemplo) mientras el while sigue trayendo páginas, y recién mostrar el juego cuando hayMas es false. datos.data.meta.total sirve para mostrar un progreso ("cargando 200 de 249").

Nombres en español

El campo names.translations trae el nombre de cada país traducido a varios idiomas, incluido español bajo la clave spa. Como ya estamos pidiendo la rama completa de names, este campo viene incluido sin tener que cambiar el response_fields. Se accede así: pais.names.translations.spa.common. No todos los países tienen traducción cargada, así que conviene un fallback a pais.names.common cuando no exista.

04 — Lógica del juego

Función: generarOpciones()

Cada pregunta necesita cuatro opciones: el país correcto y tres incorrectos elegidos al azar. Para evitar repeticiones usamos includes() antes de agregar cada opción. Al final mezclamos el array para que la respuesta correcta no esté siempre en el mismo lugar.

generar opciones
function generarOpciones() { // Array donde guardaremos las opciones let opciones = []; // Primero agregamos la respuesta correcta opciones.push(paisCorrecto.names.common); // Completamos con 3 países incorrectos al azar while (opciones.length < 4) { let random = Math.floor(Math.random() * paises.length); let nombrePais = paises[random].names.common; // Evitar duplicados if (!opciones.includes(nombrePais)) { opciones.push(nombrePais); } } // Mezclar para que la correcta no esté siempre en la misma posición opciones.sort(() => Math.random() - 0.5); // → acá iría: mostrarBotones(opciones) return opciones; }

La función retorna el array ya mezclado. Los pasos siguientes, mostrar los botones en el HTML y manejar los clics, se cubren en las funciones pendientes.

05 — Trabajo pendiente

Funciones por implementar

Lo que vimos hasta acá es la base: traer los datos y preparar las opciones. Para tener un juego completo y funcional falta desarrollar el resto de la lógica:

⚠ Funciones faltantes

  • mostrarBotones(opciones) — renderizar las 4 opciones como elementos del DOM
  • mostrarPaisAleatorio() — elegir un país al azar y mostrar su bandera en pantalla
  • verificarRespuesta(opcion) — comparar la opción elegida con la respuesta correcta
  • mostrarResultado(correcto) — feedback visual: verde/rojo según la respuesta
  • actualizarPuntaje() — sumar puntos cuando el jugador acierta
  • perderVida() — restar una vida y controlar si el juego termina
  • verificarGameOver() — detectar cuando las vidas llegan a cero
  • mostrarPantallaFinal() — mostrar el puntaje final y opción de reiniciar
  • reiniciarJuego() — resetear vidas, puntaje y empezar de nuevo
  • bloquearBotones() — evitar múltiples clicks durante la transición entre rondas

Además del HTML y CSS base: el esqueleto visual del juego con el contenedor de la bandera, los botones, el marcador de puntaje y vidas, y las transiciones entre estados (jugando → resultado → siguiente ronda → game over).
