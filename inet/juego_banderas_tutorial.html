Cómo construir un juego de adivinar banderas usando la API de REST Countries y manipulación del DOM.

01 — Objetivo

¿Qué vamos a construir?

Un juego donde se muestra una bandera aleatoria y el jugador debe elegir el nombre correcto entre cuatro opciones. Cada ronda la bandera y las opciones cambian. El jugador tiene vidas y acumula puntos.

02 — La API

REST Countries

Vamos a usar la API pública de REST Countries. No requiere registro ni API key. El endpoint que nos interesa es el siguiente:

GET https://restcountries.com/v3.1/all?fields=name,flags

Pedimos solo los campos name y flags para no traer datos innecesarios. Esto hace la respuesta mucho más liviana.

Formato de la respuesta

Cada elemento del array que devuelve la API tiene esta estructura:

respuesta JSON — un país
{ "flags": { "png": "https://flagcdn.com/w320/uy.png", "svg": "https://flagcdn.com/uy.svg", "alt": "Descripción de la bandera..." }, "name": { "common": "Uruguay", "official": "Oriental Republic of Uruguay", "nativeName": { "spa": { "official": "República Oriental del Uruguay", "common": "Uruguay" } } } }

¿Cómo accedo a cada dato?

Si tenemos un país guardado en la variable pais, accedemos a sus propiedades con punto:

Expresión→Resultado
pais.name.common→"Uruguay"
pais.name.official→"Oriental Republic of Uruguay"
pais.flags.png→"https://flagcdn.com/w320/uy.png"
pais.flags.svg→"https://flagcdn.com/uy.svg"
pais.flags.alt→"The flag of Uruguay is..."
03 — JavaScript

Función: cargarPaises()

El primer paso es traer todos los países desde la API al arrancar el juego. Usamos async/await para manejar la operación asincrónica de forma clara. Una vez que tenemos los datos, llamamos a la siguiente función que los usa.

cargar paises función
// Variable global donde guardamos todos los países let paises = []; let paisCorrecto; async function cargarPaises() { try { // fetch hace la petición a la API const respuesta = await fetch("https://restcountries.com/v3.1/all?fields=name,flags"); // Convertimos la respuesta a JSON const datos = await respuesta.json(); // Guardamos los países en la variable global paises = datos; console.log("Países cargados:", paises.length); // Una vez cargados, mostramos la primera pregunta mostrarPaisAleatorio(); } catch (error) { console.log("Error al cargar países:", error); } }

El try/catch nos permite manejar errores de red o de la API sin romper la aplicación. Si algo falla, lo mostramos en consola.

04 — Lógica del juego

Función: generarOpciones()

Cada pregunta necesita cuatro opciones: el país correcto y tres incorrectos elegidos al azar. Para evitar repeticiones usamos includes() antes de agregar cada opción. Al final mezclamos el array para que la respuesta correcta no esté siempre en el mismo lugar.

generar opciones
function generarOpciones() { // Array donde guardaremos las opciones let opciones = []; // Primero agregamos la respuesta correcta opciones.push(paisCorrecto.name.common); // Completamos con 3 países incorrectos al azar while (opciones.length < 4) { let random = Math.floor(Math.random() * paises.length); let nombrePais = paises[random].name.common; // Evitar duplicados if (!opciones.includes(nombrePais)) { opciones.push(nombrePais); } } // Mezclar para que la correcta no esté siempre en la misma posición opciones.sort(() => Math.random() - 0.5); // → acá iría: mostrarBotones(opciones) return opciones; }

La función retorna el array ya mezclado. Los pasos siguientes, mostrar los botones en el HTML y manejar los clics, se cubren en las funciones pendientes.

05 — Trabajo pendiente

Funciones por implementar

Lo que vimos hasta acá es la base: traer los datos y preparar las opciones. Para tener un juego completo y funcional falta desarrollar el resto de la lógica:

⚠ Funciones faltantes

  • mostrarBotones(opciones) — renderizar las 4 opciones como elementos del DOM
  • mostrarPaisAleatorio() — elegir un país al azar y mostrar su bandera en pantalla
  • verificarRespuesta(opcion) — comparar la opción elegida con la respuesta correcta
  • mostrarResultado(correcto) — feedback visual: verde/rojo según la respuesta
  • actualizarPuntaje() — sumar puntos cuando el jugador acierta
  • perderVida() — restar una vida y controlar si el juego termina
  • verificarGameOver() — detectar cuando las vidas llegan a cero
  • mostrarPantallaFinal() — mostrar el puntaje final y opción de reiniciar
  • reiniciarJuego() — resetear vidas, puntaje y empezar de nuevo
  • bloquearBotones() — evitar múltiples clicks durante la transición entre rondas

Además del HTML y CSS base: el esqueleto visual del juego con el contenedor de la bandera, los botones, el marcador de puntaje y vidas, y las transiciones entre estados (jugando → resultado → siguiente ronda → game over).
