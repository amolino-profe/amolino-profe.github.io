Estructura del proyecto

⚠️

Bootstrap y estilos: los ejemplos de esta guía no incluyen CSS propio para mantener el foco en la lógica MVC. Se recomienda agregar Bootstrap u otro framework de estilos a las vistas. El controlador y el modelo no deben contener ningún estilo.

Prerequisito

Tabla productos

Se asume que la base de datos y la conexión PDO ya están configuradas (ver guía anterior). Solo es necesario crear la tabla productos con la siguiente estructura:

Campo Tipo Atributos
id INT PKAUTO_INCREMENT

precio DECIMAL(10,2)

categoria VARCHAR(100)
💡

Cargar al menos 2 o 3 productos manualmente desde phpMyAdmin antes de probar el listado, así se puede verificar que la consulta funciona correctamente.

Capa de datos

Modelo — Producto.php

El Modelo concentra toda la interacción con la base de datos. Para este ABM completo expone cinco métodos: listar(), crear(), buscarPorId(), actualizar() y eliminar(). En esta guía trabajamos los dos primeros; los demás están disponibles para la actividad.

M
modelos/

Producto.php — clase completa

El constructor recibe la conexión PDO. listar() devuelve todos los registros como array asociativo usando fetchAll(). crear() usa un prepared statement con cinco parámetros ?. Los métodos de edición y eliminación ya están implementados como punto de partida para la actividad.

modelos/Producto.php 
<?php

class Producto {

    private $conn;

    public function __construct($conexion) {
        $this->conn = $conexion;
    }

    // ── READ: devuelve todos los productos ──────────────────
    public function listar() {
        $sql = "SELECT * FROM productos";
        return $this->conn
            ->query($sql)
            ->fetchAll(PDO::FETCH_ASSOC);
    }

    // ── CREATE: inserta un nuevo producto ───────────────────
    public function crear(
        $nombre, $descripcion,
        $precio, $cantidad, $categoria
    ) {
        $sql = "INSERT INTO productos
                (nombre, descripcion, precio, cantidad, categoria)
                VALUES (?, ?, ?, ?, ?)";

        $stmt = $this->conn->prepare($sql);

        return $stmt->execute([
            $nombre, $descripcion,
            $precio, $cantidad, $categoria
        ]);
    }

    // ── READ ONE: busca por id (para editar) ────────────────
    public function buscarPorId($id) {
        $sql  = "SELECT * FROM productos WHERE id = ?";
        $stmt = $this->conn->prepare($sql);
        $stmt->execute([$id]);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }

    // ── UPDATE ──────────────────────────────────────────────
    public function actualizar(
        $id, $nombre, $descripcion,
        $precio, $cantidad, $categoria
    ) {
        $sql = "UPDATE productos
                SET nombre=?, descripcion=?,
                    precio=?, cantidad=?, categoria=?
                WHERE id=?";

        $stmt = $this->conn->prepare($sql);

        return $stmt->execute([
            $nombre, $descripcion,
            $precio, $cantidad, $categoria, $id
        ]);
    }

    // ── DELETE ──────────────────────────────────────────────
    public function eliminar($id) {
        $sql  = "DELETE FROM productos WHERE id=?";
        $stmt = $this->conn->prepare($sql);
        return $stmt->execute([$id]);
    }
}
💡

PDO::FETCH_ASSOC devuelve cada fila como un array donde las claves son los nombres de las columnas, por ejemplo $producto["nombre"]. Es más legible que FETCH_NUM.

Operación 1 · READ

Listar productos

El flujo de listado es el más simple: el controlador pide al modelo todos los registros, los guarda en $productos y carga la vista, que los recorre con un foreach.

C
controladores/  ·  Controlador

listar.php

Tres líneas de lógica real: incluye la conexión, instancia el modelo y obtiene el array de productos. Luego incluye la vista para renderizar.

controladores/listar.php 
<?php

require_once "../config/database.php";
require_once "../modelos/Producto.php";

$modelo    = new Producto($conn);
$productos = $modelo->listar();

require_once "../vistas/productos/listar.view.php";
V
vistas/productos/  ·  Vista

listar.view.php

La vista recorre $productos con foreach y construye una fila de tabla por cada resultado. En la columna Acciones ya están los enlaces para editar y eliminar, pasando el id como parámetro GET — listos para cuando se implemente esa funcionalidad.

Operación 2 · CREATE

Crear un nuevo producto

El flujo de alta tiene dos momentos: la primera visita muestra el formulario vacío; cuando se envía con POST, el controlador llama al modelo para insertar y redirige al listado con header().

C
controladores/  ·  Controlador

crear.php

Si la petición es POST, extrae los campos de $_POST, llama a $modelo->crear() y redirige al listado. Si es GET (primera visita), simplemente carga el formulario vacío.

controladores/crear.php 
<?php

require_once "../config/database.php";
require_once "../modelos/Producto.php";

$modelo = new Producto($conn);

if ($_SERVER["REQUEST_METHOD"] == "POST") {

    $nombre      = $_POST["nombre"];
    $descripcion = $_POST["descripcion"];
    $precio      = $_POST["precio"];
    $cantidad    = $_POST["cantidad"];
    $categoria   = $_POST["categoria"];

    $modelo->crear(
        $nombre, $descripcion,
        $precio, $cantidad, $categoria
    );

    header("Location: listar.php");
    exit();
}

require_once "../vistas/productos/crear.view.php";
↩️

header("Location: ...") redirige al navegador a otra URL. El exit() es imprescindible inmediatamente después para que PHP no continúe ejecutando código luego de la redirección.

Actividad

Desafío — Completar el ABM

El Modelo ya tiene implementados los métodos buscarPorId(), actualizar() y eliminar(). Los controladores editar.php y eliminar.php y la vista editar.view.php están en la estructura pero vacíos. La tarea es completarlos siguiendo el mismo patrón visto.

⚡ Para resolver de forma autónoma

  • eliminar.php — Controlador que lee $_GET["id"], llama a $modelo->eliminar($id) y redirige al listado con header(). No necesita vista.
  • editar.php — Controlador con dos momentos: en GET usa buscarPorId() para precargar el formulario; en POST llama a actualizar() y redirige al listado.
  • editar.view.php — Formulario similar al de crear, pero con los campos precargados usando value="<?= $producto['campo'] ?>" y un input hidden con el id.
  • Estilos — Agregar Bootstrap (CDN) a todas las vistas para mejorar la presentación de tablas, botones y formularios.
🔑

Pista para editar: el id del producto tiene que viajar desde el listado (como parámetro GET) hasta el formulario, y desde el formulario de regreso al controlador (como campo hidden en el POST). Sin él, el UPDATE no sabe qué fila modificar.
