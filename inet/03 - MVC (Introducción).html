Patrón MVC en PHP
Registro de Personas

Implementación paso a paso del patrón Modelo–Vista–Controlador aplicado a un formulario de registro con PDO y MySQL.

Concepto

¿Qué es el patrón MVC?

MVC (Modelo–Vista–Controlador) es un patrón de arquitectura que separa una aplicación web en tres capas con responsabilidades bien definidas. Esto facilita el mantenimiento, la reutilización del código y la colaboración en equipo.

Componente Descripción Rol en PHP / MySQL
Modelo Maneja datos y lógica de negocio. Conexión a MySQL, consultas CRUD.
Vista Presenta la interfaz al usuario. HTML/CSS para renderizar datos.
Controlador Procesa entradas y coordina flujo. Enruta peticiones HTTP, actualiza modelo y selecciona vista.

Prerequisito

Base de datos y tabla

Antes de comenzar, es necesario tener creada la base de datos registro y la tabla personas. Esto es parte del trabajo previo con MySQL, que se da por conocido o se cubre en material separado.

⚠️

Recordatorio: la tabla personas debe tener al menos los campos id (INT, PK, AUTO_INCREMENT), nombre (VARCHAR), email (VARCHAR) y password (VARCHAR).

Implementación

Paso a paso

01
config/

Conexión a la base de datos — database.php

Este archivo establece la conexión usando PDO (PHP Data Objects). PDO es la forma recomendada de conectarse a bases de datos en PHP porque permite usar prepared statements y es independiente del motor de base de datos.

El atributo ERRMODE_EXCEPTION hace que cualquier error de base de datos lance una excepción, facilitando la detección de problemas.

config/database.php 
<?php

$host   = "localhost";
$dbname = "registro";
$user   = "root";
$pass   = "";

$conn = new PDO(
    "mysql:host=$host;dbname=$dbname;charset=utf8",
    $user,
    $pass
);

$conn->setAttribute(
    PDO::ATTR_ERRMODE,
    PDO::ERRMODE_EXCEPTION
);
💡

Nota: $pass = "" es válido en entornos locales (XAMPP/LAMP). En producción siempre usar contraseña.

02
modelos/  ·  Modelo

Clase Persona — Persona.php

El Modelo encapsula toda la lógica relacionada con los datos. En este caso, la clase Persona recibe la conexión PDO y expone un método guardar() que inserta un registro en la base de datos.

Se usa un prepared statement con ? como marcadores de posición. Esto protege contra inyecciones SQL. La contraseña se almacena hasheada con password_hash(), nunca en texto plano.

modelos/Persona.php 
<?php

class Persona {

    private $conn;

    public function __construct($conexion) {
        $this->conn = $conexion;
    }

    public function guardar($nombre, $email, $password) {

        $sql = "INSERT INTO personas
                (nombre, email, password)
                VALUES (?, ?, ?)";

        $stmt = $this->conn->prepare($sql);

        return $stmt->execute([
            $nombre,
            $email,
            password_hash($password, PASSWORD_DEFAULT)
        ]);
    }
}
🔒

password_hash() aplica un algoritmo seguro (bcrypt por defecto). Nunca guardar contraseñas en texto plano.

03
controladores/  ·  Controlador

Lógica de registro — registro.php

El Controlador es el intermediario: recibe los datos del formulario, los pasa al Modelo y decide qué mostrar. Incluye la conexión y el modelo con require_once. Al final, carga la Vista.

La verificación $_SERVER["REQUEST_METHOD"] == "POST" asegura que el procesamiento solo ocurra cuando el formulario fue enviado, no en una visita directa.

controladores/registro.php 
<?php

require_once "../config/database.php";
require_once "../modelos/Persona.php";

$mensaje = "";

if ($_SERVER["REQUEST_METHOD"] == "POST") {

    $nombre   = $_POST["nombre"];
    $email    = $_POST["email"];
    $password = $_POST["password"];

    $persona = new Persona($conn);

    $ok = $persona->guardar(
        $nombre,
        $email,
        $password
    );

    if ($ok) {
        $mensaje = "Usuario registrado correctamente";
    } else {
        $mensaje = "Error al registrar";
    }
}

require_once "../vistas/registro.view.php";
04
vistas/  ·  Vista

Formulario HTML — registro.view.php

La Vista es HTML puro con una pequeña inserción PHP para mostrar el mensaje de resultado. No contiene lógica de negocio; solo presenta datos.

El formulario usa method="POST" sin action, lo que envía los datos al mismo controlador que cargó la vista. La variable $mensaje viene definida por el Controlador.

vistas/registro.view.php 
<!DOCTYPE html>
<html>
<head>
    <title>Registro</title>
</head>
<body>

<form method="POST">

    <label>Nombre</label><br>
    <input type="text" name="nombre" required>
    <br><br>

    <label>Email</label><br>
    <input type="email" name="email" required>
    <br><br>

    <label>Password</label><br>
    <input type="password" name="password" required>
    <br><br>

    <button type="submit">Registrarse</button>

</form>

<p><?= $mensaje ?></p>

</body>
</html>
05
raíz del proyecto

Punto de entrada — index.php

El archivo index.php en la raíz actúa como punto de entrada único. Su función es simplemente delegar al controlador correspondiente. En proyectos más grandes aquí viviría el enrutador (router).

index.php 
<?php

require_once "controladores/registro.php";
🌐

Al acceder a http://localhost/registro/ el servidor sirve index.php automáticamente, que carga el controlador, que a su vez carga la vista.

Resumen

¿Qué hicimos?

Al separar responsabilidades en tres capas, el código es más fácil de modificar y extender. Agregar un nuevo campo al formulario solo requiere tocar cada capa en su lugar correspondiente.

⚙️

Config

Conexión PDO reutilizable. Si cambia el servidor, solo se edita aquí.

🗄️

Modelo

Encapsula la consulta SQL y el hash de contraseña. La lógica de datos en un solo lugar.

🔀

Controlador

Coordina el flujo: recibe POST, usa el modelo, pasa resultado a la vista.

🖥️

Vista

Solo HTML. No sabe nada de la base de datos; solo muestra lo que el controlador le pasa.
