DDL

Data Definition Language

El DDL se usa para definir y modificar la estructura de la base de datos: tablas, columnas, restricciones y relaciones. No toca los datos, solo el esquema.

Flujo de trabajo

El diseño siempre parte del modelo conceptual (MER) y lógico (MR), y termina en sentencias SQL.

flujo
Requerimientos  →  MER  →  Modelo Relacional  →  DDL  →  Base de Datos
💡
Los nombres de tablas, columnas y restricciones se derivan directamente del modelo lógico.
DDL

CREATE TABLE

CREATE TABLE — Sintaxis

sintaxis
CREATE TABLE <nombre_tabla> (
    <columna_1>  <tipo_dato>,
    <columna_2>  <tipo_dato>,
    -- ... más columnas
    <columna_n>  <tipo_dato>
);

Ejemplo — Tabla Persona

CREATE TABLE Persona (
    PersonaID   INT,
    Apellido    VARCHAR(100),
    Nombre      VARCHAR(100),
    Direccion   VARCHAR(255),
    Ciudad      VARCHAR(255)
);
📝
Versión básica sin restricciones. Funciona, pero acepta valores NULL en todos los campos.
CREATE TABLE Persona (
    id_usuario  INT           PRIMARY KEY,
    cedula      INT           NOT NULL UNIQUE,
    nombre      VARCHAR(20)   NOT NULL,
    apellido    VARCHAR(20)   NOT NULL,
    calle       VARCHAR(50),
    nro_puerta  VARCHAR(10),
    apto        VARCHAR(5)
);
✅
Versión robusta: la clave primaria garantiza unicidad e impide NULL. La cédula también es única.
CREATE TABLE Producto (
    id_producto   INT            PRIMARY KEY,
    codigo        VARCHAR(20)    NOT NULL UNIQUE,
    descripcion   VARCHAR(200)   NOT NULL,
    precio        FLOAT          NOT NULL,
    stock         INT            NOT NULL,
    categoria     VARCHAR(50)
);
💡
Otro ejemplo con campos numéricos. El campo categoria es opcional (acepta NULL).

Tipos de Datos

El tipo de dato define qué clase de valores puede almacenar una columna y cuánto espacio ocupa.

CategoríaTiposUso típico
Numéricos enterosINT, BIGINT, SMALLINTIDs, cantidades, edades
Numéricos decimalesFLOAT, DOUBLE, DECIMAL(p,s)Precios, coordenadas
Texto fijoCHAR(n)Códigos de longitud fija (CI, ISO...)
Texto variableVARCHAR(n)Nombres, descripciones
Texto largoTEXTComentarios, contenido libre
Fecha y horaDATE, TIME, DATETIMEFechas, horarios, timestamps
BooleanoBOOLEAN / TINYINT(1)Flags activo/inactivo
💡
CHAR vs VARCHAR: CHAR(8) siempre ocupa 8 caracteres (ideal para cédulas). VARCHAR(100) solo ocupa lo necesario (ideal para nombres).

DDL

FOREIGN KEY

FOREIGN KEY — Ejemplo Jugador / Club

Un jugador juega en un club. La tabla Juega relaciona ambas entidades usando claves foráneas.

▸ Jugador

CREATE TABLE Jugador (
    documento  CHAR(8)       PRIMARY KEY,
    nombre     VARCHAR(100)  NOT NULL,
    direccion  VARCHAR(200)
);

▸ Club

CREATE TABLE Club (
    nombre     VARCHAR(100)  PRIMARY KEY,
    fundacion  DATE,
    ciudad     VARCHAR(100)
);

▸ Juega (tabla relación)

CREATE TABLE Juega (
    documento     CHAR(8)       NOT NULL,
    nombre_club   VARCHAR(100)  NOT NULL,
    nro_camiseta  INT           NOT NULL,

    PRIMARY KEY (documento),
    FOREIGN KEY (documento)   REFERENCES Jugador(documento),
    FOREIGN KEY (nombre_club) REFERENCES Club(nombre)
);
⚠️
Para ser FOREIGN KEY, el campo en la tabla padre debe ser PRIMARY KEY o UNIQUE. Si no, el motor rechazará la sentencia.

Ejemplo adicional — Pedido / Cliente

Un caso más cotidiano: pedidos que referencian clientes y productos.

CREATE TABLE Cliente (
    id_cliente  INT           PRIMARY KEY,
    nombre      VARCHAR(100)  NOT NULL,
    email       VARCHAR(100)  UNIQUE
);

CREATE TABLE Pedido (
    id_pedido   INT   PRIMARY KEY,
    fecha       DATE  NOT NULL,
    id_cliente  INT   NOT NULL,
    total       FLOAT,

    FOREIGN KEY (id_cliente) REFERENCES Cliente(id_cliente)
);
DDL

ALTER TABLE & DROP

ALTER TABLE

Permite modificar la estructura de una tabla ya existente sin recrearla.

-- Agregar una columna nueva
ALTER TABLE Persona
    ADD telefono VARCHAR(20);

-- Agregar con valor por defecto
ALTER TABLE Producto
    ADD activo BOOLEAN DEFAULT TRUE;
-- Eliminar una columna
ALTER TABLE Persona
    DROP COLUMN telefono;

-- Ojo: esto es irreversible si no hay backup
⚠️
Eliminar una columna borra todos sus datos. No hay deshacer sin backup.
-- Agregar una FK después de crear la tabla
ALTER TABLE Pedido
    ADD CONSTRAINT FK_Pedido_Cliente
    FOREIGN KEY (id_cliente)
    REFERENCES Cliente(id_cliente);
-- Cambiar tipo o restricción de una columna
ALTER TABLE Persona
    MODIFY nombre VARCHAR(150) NOT NULL;

ALTER TABLE Producto
    MODIFY precio DECIMAL(10,2) NOT NULL;

DROP TABLE

Elimina una tabla completa, con todos sus datos y estructura.

DROP TABLE Juega;          -- primero las tablas con FK
DROP TABLE Jugador;
DROP TABLE Club;

-- Versión segura (no falla si no existe)
DROP TABLE IF EXISTS Jugador;
⚠️
No se puede eliminar una tabla que tenga FOREIGN KEY apuntando a ella desde otra tabla activa. Primero hay que eliminar la tabla dependiente.
DML

Data Manipulation Language

El DML opera sobre la instancia de la base de datos: inserta, modifica y elimina filas (tuplas) en las tablas. Trabaja sobre los datos, no sobre la estructura.

Los tres comandos principales del DML:

INSERT Agrega nuevas filas
UPDATE Modifica filas existentes
DELETE Elimina filas existentes
🔑
DDL modifica el esquema (estructura). DML modifica la instancia (datos).
DML

INSERT

INSERT — Sintaxis

sintaxis
INSERT INTO <nombre_tabla> (col1, col2, ..., colN)
VALUES (val1, val2, ..., valN);

Ejemplos de INSERT

-- Insertar un elemento en la tabla elementos
INSERT INTO elementos (simbolo, nombre)
VALUES ('H', 'Hidrógeno');

-- Insertar una persona
INSERT INTO Persona (id_usuario, cedula, nombre, apellido, calle)
VALUES (1, 12345678, 'Ana', 'García', 'Av. Italia');
-- Insertar varias filas en una sola sentencia
INSERT INTO elementos (simbolo, nombre)
VALUES
    ('H',  'Hidrógeno'),
    ('He', 'Helio'),
    ('Li', 'Litio'),
    ('C',  'Carbono');
⚡
Insertar múltiples filas en una sola sentencia es más eficiente que hacer varios INSERT individuales.
-- Si se omite una columna, queda NULL en ese campo
INSERT INTO Persona (id_usuario, cedula, nombre, apellido)
VALUES (2, 87654321, 'Luis', 'Pérez');

-- Equivalente a:
INSERT INTO Persona (id_usuario, cedula, nombre, apellido, calle, nro_puerta, apto)
VALUES (2, 87654321, 'Luis', 'Pérez', NULL, NULL, NULL);
⚠️
Si una columna tiene NOT NULL y la omitís, el motor rechazará la inserción con un error.
-- Sin especificar columnas: hay que respetar el orden de la tabla
INSERT INTO elementos
VALUES ('O', 'Oxígeno');

-- ❌ Práctica NO recomendada: si cambia el orden de columnas, falla
💡
Buena práctica: especificar siempre las columnas en el INSERT. Hace el código más legible y resistente a cambios de esquema.
DML

UPDATE

UPDATE — Sintaxis

sintaxis
UPDATE <nombre_tabla>
SET col1 = val1, col2 = val2, ..., colN = valN
WHERE <predicado>;
ℹ️
Todas las filas que cumplan el predicado serán modificadas simultáneamente. Si el predicado coincide con 100 filas, las 100 se actualizan.

Ejemplos de UPDATE

-- Corregir tilde en nombre
UPDATE elementos
SET nombre = 'Hidrógeno'
WHERE nombre = 'Hidrogeno';

-- Actualizar dirección de una persona por su ID
UPDATE Persona
SET calle = 'Bvar. Artigas', nro_puerta = '1234'
WHERE id_usuario = 1;
-- Modificar precio y stock de un producto
UPDATE Producto
SET precio = 299.90,
    stock  = stock + 50
WHERE id_producto = 5;

-- Aplicar descuento del 10% a una categoría
UPDATE Producto
SET precio = precio * 0.90
WHERE categoria = 'Electrónica';
-- Marcar como inactivos los productos con poco stock
UPDATE Producto
SET activo = FALSE
WHERE stock BETWEEN 0 AND 5;

-- Actualizar pedidos de un período
UPDATE Pedido
SET estado = 'archivado'
WHERE fecha BETWEEN '2023-01-01' AND '2023-12-31';
-- Asignar ciudad por defecto a quienes no tienen
UPDATE Persona
SET Ciudad = 'Montevideo'
WHERE Ciudad IS NULL;

-- Quitar un email (poner NULL) para un usuario
UPDATE Cliente
SET email = NULL
WHERE id_cliente = 10;

Predicados en el WHERE

El predicado filtra qué filas se verán afectadas. Puede combinar varias condiciones:

= < > <>Comparación simple
BETWEENRango de valores
LIKEExpresión regular
IS NULLComparar con null
IS NOT NULLNo es null
AND / OR / NOTLógica booleana
IN (...)Lista de valores
-- Ejemplos combinados
UPDATE Persona
SET activo = FALSE
WHERE apellido LIKE 'Garc%' AND Ciudad IS NOT NULL;

UPDATE Producto
SET precio = precio * 1.05
WHERE id_producto IN (3, 7, 12);
DML

DELETE

DELETE — Sintaxis

sintaxis
DELETE FROM <nombre_tabla>
WHERE <predicado>;

Ejemplos de DELETE

-- Eliminar un elemento por su símbolo
DELETE FROM elementos
WHERE simbolo = 'H';

-- Eliminar un cliente por ID
DELETE FROM Cliente
WHERE id_cliente = 7;
-- Eliminar movimientos anteriores a una fecha
DELETE FROM movimientos
WHERE fecha_mov < '2002-05-15';

-- Eliminar pedidos del año pasado
DELETE FROM Pedido
WHERE fecha BETWEEN '2023-01-01' AND '2023-12-31';
-- Eliminar productos inactivos sin stock
DELETE FROM Producto
WHERE activo = FALSE AND stock = 0;

-- Eliminar personas sin dirección ni ciudad
DELETE FROM Persona
WHERE calle IS NULL AND Ciudad IS NULL;
⚠ CUIDADO

UPDATE y DELETE sin WHERE

El predicado vacío aplica a todas las filas

Si omitís el WHERE, el comando afecta absolutamente todas las filas de la tabla.

❌ PELIGROSO

-- Sobreescribe TODOS los símbolos
UPDATE elementos
SET simbolo = 'H';

-- Elimina TODA la tabla
DELETE FROM elementos;

✅ CORRECTO

-- Solo actualiza el que corresponde
UPDATE elementos
SET simbolo = 'H'
WHERE nombre = 'Hidrógeno';

-- Solo elimina el que corresponde
DELETE FROM elementos
WHERE simbolo = 'H';
🔥
Regla de oro: antes de ejecutar un UPDATE o DELETE, probá primero el predicado con un SELECT para verificar qué filas afectaría. Así evitás pérdidas de datos irreversibles.
tip: probar primero
-- 1. Primero verificar con SELECT
SELECT * FROM elementos
WHERE simbolo = 'H';

-- 2. Si el resultado es el esperado, entonces ejecutar
DELETE FROM elementos
WHERE simbolo = 'H';

Resumen de comandos

CREATE TABLE

Crea una tabla nueva con columnas y restricciones

ALTER TABLE

Modifica la estructura de una tabla existente

DROP TABLE

Elimina una tabla y todos sus datos

INSERT

Agrega nuevas filas a una tabla

UPDATE

Modifica filas existentes que cumplen un predicado

DELETE

Elimina filas que cumplen un predicado
